# Установка docker и запуск контейнера для лаб по МКЭ

## Условия

ОС Убунту, внимательность

## Установка Docker

### Установи Docker (если не установлен):

    sudo apt update
    sudo apt install ca-certificates curl
    sudo install -m 0755 -d /etc/apt/keyrings
    sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
    sudo chmod a+r /etc/apt/keyrings/docker.asc

    sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
    Types: deb
    URIs: https://download.docker.com/linux/ubuntu
    Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
    Components: stable
    Architectures: $(dpkg --print-architecture)
    Signed-By: /etc/apt/keyrings/docker.asc
    EOF

    sudo apt update

    sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

После установки проверь, что процесс Docker выполняется:

    sudo systemctl status docker

Если предыдущая команда показала, что Docker "down", запусти его вручную: 

    sudo systemctl start docker

Убедись, что установка прошла штатно, запустив образ hello-world:

    sudo docker run hello-world

Создай роль для пользователей Docker и добавьте её вашему пользователю:

    sudo groupadd docker
    sudo usermod -aG docker $USER
    newgrp docker

После этого может потребоваться перезагрузка ПК (обычно не требуется).

## Сборка и запуск "контейнера"

### 1. Запуск контейнера из уже готового образа

Скачай контейнер:
1. Для этого нужно авторизоваться в yandex cloud (там хранится образ)

        curl -sSL https://storage.yandexcloud.net/yandexcloud-yc/install.sh | bash

2. Перезапустите консоль\терминал

3. Эта команда запишет твой токен в конфиг, после ввода нужно перейти по ссылке из консоли, выбрать свой аккаунт, появится токен, его нужно вставить в консоль.

        yc init

Теперь можно скачивать образ из облака с помощью, например, CLI.

    docker pull cr.yandex/crpdtgu6t719a7cssr10/fenics-env:v0.1

#### Контейнер содержит в себе все необходимые библиотеки для работы с gmesh, fenics-legacy и streamlit

### Запуск с помощью VS Code
#### Для VS Code уже сконфигурирован файл *devcontainers.json* в этом репозитории.

При запуске через VS Code должна быть открыта папка проекта.

Запуск через VS Code происходит выполнением команды Reopen in container:

CTRL+SHIFT+P -> Dev containers: Reopen in containers

### Запуск с помощью CLI

#### Запуск консольной командой
При запуске через консоль нужно находиться в корневой директории репозитория.

    docker run --rm -it \
      -v "$(pwd)":/workspace \
      -w /workspace \
      cr.yandex/crpdtgu6t719a7cssr10/fenics-env:v0.1

#### Запуск консольным скриптом
 
Для этого в консоли, **находясь в корневой директории проекта**, выполните команду (разрешить скрпит к выполнению):

    chmod +x run.sh

Затем запуск контейнера будет происходить с помощью скрипта:

    ./run.sh

---
Контейнер во всех случаях запускается в интерактивном режиме. Это значит, что консоль, которая запускала скрипт, станет виртуальной консолью контейнера. В таком варианте все python-скрипты запускаются командой.

Например:

    python3 solver/run_batch.py



## Сборка кастомного образа

На основе скачанного образа можно создать свой, добавив новые библиотеки, инструменты и т.д.

### Описание образа

Для создания своего образа нужно создать свой Dockerfile и указать базовый образ:

    FROM cr.yandex/crpdtgu6t719a7cssr10/fenics-env:v0.1

Затем ниже указать уже необходимые изменения, например установку библиотеки python:
    
    RUN python3 -m pip install --no-cache-dir *some-lib-name*


### Сборка образа

После описания образа в Dockerfile, образ нужно собрать, чтобы он попал в регистр Docker. Это выполняется командой build. Команда выполняется строго в директории, в которой находится созданный тобой Dockerfile.

    docker build -t *your-image-name* .



##  Работа с контейнером

### Рабочая директория

Внутри контейнера рабочей директорией является /workspace. Все описанные способы запуска контейнера монтируют активную [ $(pwd) ] директорию в качестве /workspace. Таким образом, при работе в контейнере, все внесённые в пределах этой директории изменения будут сделаны в основной файловой системе хоста.   

### Streamlit

Для запуска streamlit выполните команду:

    streamlit run app.py

Затем открыть на хосте (возможен другой порт, кроме как 8501; смотреть в консоли после запуска):

    http://127.0.0.1:8501


## Работа с программой для лабораторной работы 2

Приведённые ниже команды отвечают за полный цикл "программы". Запускают построение сеток, затем вычисляют результат и создают "картинки".

    python3 solver/run_batch.py
    python3 solver/build_convergence_table.py

Запустить единственный "прогон" можно скриптом

## PS 

Как вообще с этим работать? Просто скачай себе VS Code, установи докер, и нажми в vs code "Reopen in container" для этого репозитория (папки, проекта, и т.д.). Всё уже сконфигурировано и должно запуститься автоматически. Попадёшь в новое окно VS Code, с теми же файлами и подпапками, но уже внутри контейнера. Должен работать и Pylance, и IntelliSense, и даже дебаггер. Проще, наверное, некуда.

Важно понимать, что контейнер использует обособленную "программу" в качестве gmesh. Т.е. gmesh то один, как у всех, но работа с ним происходит через его исполняемый файл. 

    gmsh -2 path_to_structure_file.geo -o path_to_output_file.msh

Сделано это намеренно, потому что автор считает, что использование gmsh python API это смешивание в кучу мух, мёда, известной субстанции и пчёл.

Построение сетки - отдельная процедура. Сетка описывается в нотации gmesh, затем скриптом конвертируется в удобоваримый fenics-legacy формат. 

Хотя это и не принципиально, и автор, в силу некомпетентности в вопросе работы с сетками, может ошибаться.
